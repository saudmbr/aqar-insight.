Replace the current insecure password reset flow with a secure email-based reset flow.

Current issue:
- Users can reset password directly without verification
- This is not allowed and is a security risk

Requirements:

1) Forgot password page:
- Keep /forgot-password page
- User enters email only
- Show Arabic UI and clear messages

2) Secure reset flow:
- When user submits email:
  - Generate a secure random reset token
  - Token expiry must be 15 minutes
  - Store the token securely in the database
  - Send a password reset email containing a reset link:
    /reset-password?token=XYZ

3) Security rules:
- Do NOT reveal whether the email exists or not
- Always show a generic success message like:
  "إذا كان البريد الإلكتروني مسجلًا لدينا، فسيتم إرسال رابط استعادة كلمة المرور"
- Token must be one-time use only
- Token must expire automatically
- Prevent brute force and abuse with rate limiting

4) Reset password page:
- Create /reset-password page
- Validate token before showing the password form
- If token is invalid or expired, show an Arabic error message
- If valid, allow user to enter:
  - new password
  - confirm password

5) Password rules:
- Minimum 8 characters
- Must include letters and numbers

6) After successful reset:
- Update password securely (hashed)
- Invalidate token
- Redirect user to /login
- Show Arabic success message:
  "تم تغيير كلمة المرور بنجاح"

7) Email integration:
- Add email sending support using environment variables / secrets
- Support SMTP or an email service provider
- Put required secrets in environment variables, such as:
  SMTP_HOST
  SMTP_PORT
  SMTP_USER
  SMTP_PASS
  SMTP_FROM

8) Development mode:
- If email is not configured yet, show the reset link only in development logs
- Never expose reset links publicly in production

9) UI/UX:
- Keep the design modern, Arabic, RTL, clean, and consistent with the platform

10) Important:
- Remove any direct password reset without verification
- Make this feature production-ready and secure
